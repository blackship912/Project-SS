# Project:SS — Alpha 0.6.3

地方中古戸建て投資の物件評価・調査・学習ログを行う UI プロトタイプです。
「記録するアプリ」ではなく「使っているうちに記録されるアプリ」をコンセプトに、
物件のスコアリング・調査チェック・写真管理・比較・お気に入りなどを
HTML/CSS/JavaScript のみ（単一ファイル）で実装しています。

**Alpha 0.6.2 はセーブポイントとして別ファイルのまま保持しています。
このリポジトリの `index.html` は Alpha 0.6.3 の実験版です。**

## GitHub Pages での公開方法

1. このリポジトリの内容（`index.html` と `.nojekyll`）をリポジトリのルートに配置します。
2. GitHub の **Settings → Pages** を開きます。
3. **Source** を `Deploy from a branch` に設定し、対象ブランチ（例: `main`）とフォルダ `/ (root)` を選択します。
4. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` でアクセスできるようになります。

`.nojekyll` は GitHub Pages のデフォルトの Jekyll 処理をスキップし、
ファイルをそのまま静的配信させるためのものです。

**iPhone Safari での動作を前提に作られています。** タップ判定・スワイプ・ボトムシートの
挙動は実機の Safari で確認しながら調整しました（PC ブラウザでも一通り操作できます）。

## Alpha 0.6.3 完成版に向けた追加修正（第2弾）

1. **写真アクションシートを、発生源から広がるポップアップメニューに刷新**
   タップしたボタンの位置を起点に、そこから広がるように表示する、iOSのコンテキスト
   メニューやmacOSのポップオーバーに近い動きに作り直しました。画面の背景は暗くせず、
   ポップアップ自体だけ半透明のダークグレー（すりガラス質感）にしています。枠外を
   タップすると閉じます。「＋ 写真を追加」ボタンでは写真ライブラリ／写真を撮る／
   ファイルを選択の3項目（削除なし）、「🔄 この写真を変更」およびフルスクリーン内の
   「この写真を変更」では同じ3項目に「この写真を削除」（赤文字）を加えた4項目を
   表示します。フルスクリーンの独立した削除ボタンはこのポップアップに統合しました。

2. **写真画面の微調整**
   写真内のタグ表示をもう少し下へ、ドットインジケーターに近づけつつ、くっつかない
   ようbottomの値を調整しました。「写真を編集」ボタンの下に余白を追加し、外枠と
   くっついて見える問題を解消しています。

3. **「その他・手入力」で手入力欄が表示されないことがある不具合を修正**
   「その他・手入力」を選択した直後は、まだ手入力欄自体がDOM上に存在しないため、
   その状態で値を確定してしまうと入力内容が空にリセットされ、結果として次の再描画で
   手入力欄そのものが表示されなくなる、という不具合がありました。「このフィールドは
   いま『その他』を選択中である」という状態を専用のフラグとして保持する仕組みに変更し、
   選択した瞬間から確実に手入力欄が表示され、入力した内容も保持されるようにしています。
   これは構造・間取り・用途地域・情報源のすべてに共通で効く仕組みとして実装したので、
   今後同様のプルダウン項目を追加した場合もそのまま動作します。

4. **ハンドル操作中にSafari自体のオーバースクロール（引っ張り）が起きる不具合を修正**
   ハンドルをドラッグしている間、背後の画面までゴムのように伸びてしまう現象がありました。
   ドラッグ中のタッチイベントで明示的に既定動作を打ち消す処理を追加し、シート自身の
   みが動き、背景の画面は一切動かないようにしています。あわせて、見た目のハンドルは
   そのままの細い棒の見た目を保ちつつ、実際にタップ・ドラッグを検知する範囲を大きく
   広げました。これは共通関数を通じて、ホーム右上メニュー・メニュー詳細・基本情報編集・
   物件のステータス変更・調査項目のステータス変更のすべてに一律で適用されています。

5. **ボトムナビ「お気に入り」の文字色を修正**
   ハートのアイコンだけ赤くし、「お気に入り」という文字ラベルは他のメニューと同じ
   色に統一しました。

6. **クイックリンクで移動した際、カードの上端が隠れて見える問題を調整**
   スクロール後の余白を広げ、カードの上枠がきちんと見える位置で止まるようにしました。

## Alpha 0.6.3 完成版に向けた追加修正

Save Point 方式のため、これは新しいバージョン（0.6.4 など）ではなく、
Alpha 0.6.3 自体をそのまま磨き上げた修正です。

1. **物件追加フォームの入力が基本情報へ反映されない不具合を修正**
   原因は、基本情報編集ウィザードと新規物件追加フォームが、入力欄のIDを
   `wf_price_num` のように同じ命名規則で生成していたことでした。HTML上に
   同じIDの要素が複数存在すると `getElementById` は最初に見つかった方
   （編集ウィザード側）を返してしまうため、新規追加フォームにいくら入力しても
   実際には反映されない状態になっていました。入力欄のID生成に呼び出し元ごとの
   プレフィックス（編集ウィザードは `wf_`、新規追加フォームは `np_`）を持たせる
   共通の仕組みに直し、今後同種のフォームを追加してもこの種の衝突が起きない
   ようにしています。

2. **「その他・手入力」の共通ルールを整理**
   構造・間取り・用途地域・情報源など、プルダウンに「その他・手入力」を持つ
   項目は、選択すると手入力欄が表示され、入力内容がそのまま保存・基本情報へ
   反映されるようになっています（これは 1. の修正と地続きの内容です）。
   手入力した内容が長くなる場合に備え、基本情報の各項目は2行までの表示に
   統一し、それを超える場合はタップで全文を確認できる共通の詳細シートを
   新設しました。今後、長くなり得るどの項目にもこの仕組みをそのまま使えます。

3. **共通ハンドルのドラッグ挙動を実装し直し**
   これまでは指を離した瞬間の位置だけで開閉を判定しており、ドラッグ中に
   シート自体が指に追従して動く演出がありませんでした。今回、ドラッグ中は
   シートだけがその場で指に追従して動き、背景の画面（ホームなど）は一切動かない
   実装に変更しています。一定距離を超えて引く、またはすばやくフリックすると
   そのまま下へスライドして閉じ、閉じ方は「キャンセル」「閉じる」ボタンを押した
   時と全く同じ処理に合流します。対象はホーム右上メニュー・メニュー詳細・
   基本情報編集・物件のステータス変更・調査項目のステータス変更で、この5箇所
   すべてに同じ関数を適用し、共通仕様として統一しました。

4. **写真画面の調整**
   タグの表示が写真の内側・外側の両方に重複していたため、写真内側の表示だけを
   残しました。写真内のタグ位置は少し下へ寄せています。「写真の閲覧」という
   見出しは削除しました。

   「この写真を変更」から開くメニューは、iOS/Androidのネイティブなアクションシート
   （半透明のダークグレー・すりガラス質感・項目のグループ化・キャンセルボタンの分離）
   に似せた専用デザインで作り直しました。ブラウザの`<input type="file">`が呼び出す
   「写真ライブラリ／写真を撮る／ファイルを選択」というシステムUI自体は、
   Web技術からは中身を変更したり項目を追加したりすることができない仕様のため、
   この3項目とは別に、Project:SS側で用意したアクションシートの中に同じ3項目と
   「この写真を削除」（赤文字）を並べています。見た目をできる限りネイティブの
   雰囲気に近づけつつ、削除機能を自然に追加できる形にしています。

5. **調査項目のステータス変更UIを、物件全体のステータス変更と統一**
   これまで調査項目は「タップで変更」という補足テキスト付きのバッジ表示でしたが、
   物件全体のステータス表示（「候補 ▾」のような、値＋下向き矢印の表記）と
   同じ見た目に揃えました。

6. **ホーム画面の色味・ブランド調整**
   注目物件カードだけ他のカードよりわずかに緑がかっていた点を、白背景に統一し、
   代わりに左端にごく細いブランドカラーのアクセントバーを添える形に変更しました。
   カード全体のシャドウを少し薄く・範囲も少し狭くし、浮遊感のある軽い印象に
   調整しています。見出しの文字の太さもわずかに落とし、カード間の余白も
   気持ち広げました。

## Alpha 0.6.3 での変更点（0.6.2 からの差分）

今回のテーマは「0.6.2 の残修正」「色味調整」「物件追加機能の入口実装」です。

1. **写真の削除を「この写真を変更」メニューに統合**
   通常の写真編集画面には削除手段がなかったため、「🔄 この写真を変更」をタップした時に開く
   メニューに、写真ライブラリ／写真を撮る／ファイルを選択／区切り線／この写真を削除、という
   構成でボトムシートを追加しました。編集画面自体のレイアウトは増やしていません。
   フルスクリーン表示側の削除機能とは同じ削除処理を共有しているので、
   どちらから削除しても物件データやログの扱いは統一されています。

2. **お気に入り画面からホームへ戻れない不具合を修正**
   ボトムナビの「ホーム」ボタンが、詳細画面専用の「元いた画面に戻る」ロジックを
   誤って流用していたため、過去に「お気に入り→詳細→戻る」を一度でも経由していると、
   その後ホームボタンを押してもお気に入り画面から動けなくなる不具合がありました。
   ホームボタンは常にホーム画面へ直行するよう分離し、比較・設定画面など
   他のどの画面からでも正しくホームへ戻れることを確認しています。

3. **ホーム右上メニューの「閉じる」でメニュー一覧へ戻れるように**
   これまでは項目の詳細を閉じると完全にホーム画面まで戻ってしまっていました。
   今回、メニュー一覧のシートを閉じずに裏で開いたままにし、詳細シートをその上に
   重ねて表示する方式に変更。詳細を閉じるとメニュー一覧が自然に見える形になります。
   メニュー一覧・詳細どちらの画面も、上部のハンドルをタップ、または下にスワイプすると
   閉じられます。背景のホーム画面自体は動かず、シートだけが下に滑らかに閉じます。

4. **色味調整**
   ページ背景を `#F6FAF8`、カード背景をウォームホワイトの `#FFFEFC` に変更しました。
   文字色とのコントラスト比は計算上 15 以上あり、WCAG AA 基準（4.5 以上）を大きく
   上回っているため、可読性を落とさずに全体の印象をやわらかくしています。

5. **「＋」ボタンから新規物件を追加できるように**
   ボトムナビ中央の「＋」を押すと、物件名・住所・価格・利回り・家賃・築年数・構造・現況・
   建物面積・土地面積・間取り・駐車場・接道・用途地域・情報源・ステータスを入力できる
   シートが開きます。すべて未入力のまま登録しても構いません（物件名が空の場合は
   「無題の物件」として登録されます）。登録すると、その場で新しい物件の詳細画面が開き、
   お気に入り・ステータス変更・写真追加・メモ・調査チェックが既存物件と全く同じように
   利用できます。ホーム画面の一覧にも追加されます。

## Alpha 0.6.2 での変更点（0.6.1 からの差分）

ユーザー目線でのサマリーです。

1. **SSロゴを Design Reference（Ver.1.2 UX Prototype のロゴ）に忠実な形へ再修正**
   直前のやり取りで「盾型」「初期モックの角丸アイコン」といった別方向の修正を試しましたが、
   このチャットで最初に共有した Ver.1.2 UX Prototype 画像の左上にあるロゴ
   ——縦にやや長い六角形・緑単色・白い二重ライン・中央に「SS」——
   を正式な Design Reference として採用し、その形そのものを CSS で高品質に再現しました。
   六角形の頂点比率はリファレンス画像をピクセル単位で解析して割り出しています。
   内側の白いラインも、単純な矩形オフセットではなく六角形の中心から相似縮小した
   3層構造（緑→白→緑）にすることで、上下の辺だけが太い帯に見えてしまう問題を解消し、
   どの辺でも均等な太さの六角形ラインとして見えるようにしています。
   設定画面からは白ラインの太さ違い（標準ライン／二重ライン強調）を見比べられます。

   ホーム画面から詳細を開いた場合はこの問題が起きにくく、見過ごされていました。

   これにより、お気に入り画面から開いた場合は戻るボタンでお気に入り画面へ、

## 既知の未対応点（現時点での制約）

- データはブラウザのメモリ上にのみ保持され、リロードすると初期状態（サンプル3物件）に戻ります。
  永続化（サーバー保存やlocalStorageなど）は意図的に未実装です。新規登録した物件も同様です。
- 新規物件登録シートおよび写真変更アクションシートの「写真ライブラリ／写真を撮る／
  ファイルを選択」は、Webの技術的な制約上すべて同じ `<input type="file" accept="image/*">`
  を呼び出しています。iOS Safari がこの入力に対して自動的に出すネイティブの選択肢
  （フォトライブラリ／カメラ／ファイル）をそのまま利用しているため、Web側の実装で
  3つの動作を個別に切り分けることはできません。写真変更アクションシートの見た目自体は
  iOS/Androidのネイティブなアクションシートに似せていますが、あくまでWeb上で描画した
  独自のUIであり、OS標準のシートそのものではありません。
- 「盾ロゴ」の完全な廃止は行っておらず、設定画面のトグルとして選択肢自体は残しています
  （方針転換により見た目は Ver.1.2 UX Prototype のロゴに寄せて修正済みです）。
- 音声入力ボタン（写真メモ）は見た目上のアニメーションのみで、実際の音声認識は行いません。
- 数値入力欄は `inputmode="decimal"` を指定していますが、桁区切りカンマの自動整形などは
  行っていません（カンマ付きで入力してもそのまま保持されます）。
- 新規登録した物件はスコアが仮の初期値（各項目10点、合計50点）で登録され、
  AIコメントも「まだAIコメントがありません」という仮の文言のままです。
  実際のスコア再計算ロジックは今回のスコープに含めていません。
