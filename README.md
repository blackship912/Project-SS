# Project:SS — Alpha 0.6.1

地方中古戸建て投資の物件評価・調査・学習ログを行う UI プロトタイプです。
「記録するアプリ」ではなく「使っているうちに記録されるアプリ」をコンセプトに、
物件のスコアリング・調査チェック・写真管理・比較・お気に入りなどを
HTML/CSS/JavaScript のみ（単一ファイル）で実装しています。

**Alpha 0.6 はセーブポイントとして別ファイルのまま保持しています。
このリポジトリの `index.html` は Alpha 0.6.1 の実験版です。**

## GitHub Pages での公開方法

1. このリポジトリの内容（`index.html` と `.nojekyll`）をリポジトリのルートに配置します。
2. GitHub の **Settings → Pages** を開きます。
3. **Source** を `Deploy from a branch` に設定し、対象ブランチ（例: `main`）とフォルダ `/ (root)` を選択します。
4. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` でアクセスできるようになります。

`.nojekyll` を置いているのは、GitHub Pages のデフォルトの Jekyll 処理をスキップし、
ファイルをそのまま静的配信させるためです（アンダースコアで始まるファイル名などが
Jekyll に無視される事故を防ぎます）。

## Alpha 0.6.1 での変更点（0.6 からの差分）

1. ボトムナビの「お気に入り」アイコンを赤いハートに変更
2. 物件詳細画面で🏠ホームを押すと確実にホーム画面へ戻るよう修正
   （0.6 では `currentNav` が更新されず反応しないことがありました）
3. 「基本／調査／メモ／ログ」の横スワイプ切り替えエリアに
   写真ギャラリーと同様のドットインジケーターを追加
4. Quick Links（基本・調査・メモ・ログ）を `data-pi` 属性ベースで確実に動作するよう実装
5. スコアカード直下の平文コメントを廃止し、
   「🤖 AI要約（1行）＋ AI詳細を見る →」の導線に変更。
   タップすると AI アドバイスカードまで自動スクロールし、詳細を自動展開します
6. 写真の「閲覧」と「編集」を分離。
   通常表示は確定済みタグ・メモのプレビューのみで、
   「✎ 写真を編集」を開いたときだけタグ付け・メモ編集・写真追加/変更ができます
7. 横スワイプのアニメーションをより滑らかに調整
   （慣性フリック判定・端でのわずかな抵抗感を追加）
8. SS ロゴの盾型デザイン案を実装。設定画面から現行の角丸デザインと
   見比べて切り替えられます（採用可否は保留中の試験実装）

色味（テーマカラー）の変更、および比較機能自体の仕様は今回は変更していません。

## 注意事項

これはプロトタイプ／試作版です。データはブラウザのメモリ上にのみ保持され、
リロードすると初期状態（サンプル3物件）に戻ります。永続化やログイン機能は
意図的に実装していません。
